Well, it's official: I'm now a professional geek. OK, so I get paid to write computer programs which may or may not be the same thing :-) Either way, here's the usual laundry list of useful and/or interesting links related to my day-to-day computing.

Linux

  • I am Linux user #180140 at the Linux Counter. Eighteen years of using Linux :-)
  • I'm running Fedora 20 on a 6-core Phenom II AMD64 system with 8 GB of RAM (though I use Ubuntu at home).
  • The Fedora Forum is a useful place for help, info and advice.
  • Linux news at Linux Today and Linux Weekly News.
  • For those who care, I currently use Cinnamon as my desktop environment.

Perl

  • All things Perl.
  • Actually, the real source of Perl's power, CPAN.
  • Perl documentation at perldoc.org.
  • Perl news and stuff at the O'Reilly web site.
  • Learning the wonder that is Object-Oriented Perl?

Astronomical software

  • My day job: SCUBA-2 data reduction software. Progress and updates can be found on the Pipelines and Archives blog, while pretty pictures are on the official SCUBA-2 blog.
  • The UK Starlink project (now disbanded). Starlink software can be obtained from the JAC Starlink page.
  • The ORAC-DR data reduction pipeline and its post-processing relative PICARD.
  • A conference a year courtesy of ADASS.
  • FITS: the standard (-ish) astronomical data format.

Other software

  • The real way to make web pages look good: the W3 site for CSS, and Eric Meyer's page on CSS.
  • The real way to make your printed page look good: LATEX. Unbeatable for maths typesetting.
  • OK if you must use an office suite, then consider LibreOffice. It comes with its own PDF writer, so there's no excuse for sending .doc files by email (of course, there's rarely an excuse to send .doc files by email...). Plus it supports the Open Document Format.
  • But nothing beats Emacs for doing everything else :-)
  • My must-have apps are (in no particular order):
    • Firefox and/or Google Chrome for the web
    • Thunderbird for email
    • Emacs for editing anything
    • Xfig for graphics
    • TeX/LaTeX for documentation (+dvips/dvipdf)
    • Evince and gv for viewing PDF and PostScript files respectively
    • The Gimp for photos etc
    • GQView and Gwenview for image viewing
    • LibreOffice comes in handy for the odd Word doc, making posters and presentations for conferences

About these pages

I've hand-coded these pages mostly using Emacs (with a sprinking of Bluefish) and the W3 guides to HTML 4, XHTML and CSS. I use XHTML-1.0 with a strict DTD with CSS for layout and style. All pages validate as valid XHTML-1.0. Hopefully the pages render OK with all modern browsers: I don't have access to IE or Safari so I can't test them. Pretty simple stuff, really.
